# Инструкция: личный ассистент DeepSeek-V3

Это твой **бесплатный ИИ-ассистент в терминале** — как Claude Code, но на бесплатной
модели DeepSeek-V3 (671B) через GitHub Models. Он реально работает с твоим ПК: читает и
правит файлы, ищет по коду, выполняет команды. Мозг — в облаке, твой компьютер не грузится.

---

## 1. Что установить (один раз)

Нужен Python 3 (для Windows — с python.org, при установке галочка «Add Python to PATH»).
Дальше — одна библиотека.

### Вариант A — Windows PowerShell (как у тебя)
Открой **PowerShell** и выполни (команды вводи по одной, без строк с ` ``` `):
```powershell
cd "C:\Users\Mick Thomson\deepseek-assistant"
pip install -r requirements.txt
```

### Вариант B — WSL (Ubuntu)
```bash
cd "/mnt/c/Users/Mick Thomson/deepseek-assistant"
pip install -r requirements.txt
```

Токен GitHub уже прописан в `.env` (в git не попадает). Заменить — см. `.env.example`.

## 2. Как запускать

Важно: ассистент видит файлы и выполняет команды **в той папке, откуда запущен**. Зайди в
нужную папку и запусти.

### Windows (PowerShell)
```powershell
cd "C:\путь\к\твоему\проекту"
& "C:\Users\Mick Thomson\deepseek-assistant\assist.bat"
```
Либо прямо: `python "C:\Users\Mick Thomson\deepseek-assistant\assistant.py"`

(Чтобы запускать просто командой `assist` — добавь папку `deepseek-assistant` в переменную
среды PATH; тогда из любой папки сработает `assist`.)

### WSL (Ubuntu)
Сделай ярлык один раз:
```bash
echo "alias assist='python3 \"/mnt/c/Users/Mick Thomson/deepseek-assistant/assistant.py\"'" >> ~/.bashrc
source ~/.bashrc
```
Потом: `cd /путь/к/проекту && assist`

Откроется чат. Дальше просто пиши задачу обычными словами, как мне.

> ⚠️ Не вставляй строки вида ` ```bash ` или ` ```powershell ` — это оформление инструкции,
> а не команды. И не путай пути: в PowerShell — `C:\Users\...`, в WSL — `/mnt/c/Users/...`.

## 3. Команды чата

| Команда | Что делает |
|---|---|
| `/plan`  | включить **PLAN MODE** — ассистент только читает и выдаёт план, **ничего не меняет** |
| `/run`   | выключить PLAN MODE — разрешить правки файлов и команды |
| `/yolo`  | вкл/выкл авто-подтверждение (не спрашивать «Выполнить?» на каждое действие) |
| `/model` | показать модели и какая отвечает **сейчас** |
| `/remember <текст>` | запомнить факт о себе **навсегда** (во всех будущих разговорах) |
| `/memory` | показать, что ассистент о тебе помнит |
| `/forget` | стереть всю память |
| `/sessions` | список прошлых разговоров |
| `/resume [№]` | продолжить прошлый разговор (без номера — последний) |
| `/new`   | начать новый разговор |
| `/reset` | очистить текущий разговор |
| `/help`  | список команд |
| `/exit`  | выход |

### Память и сессии (как у больших ассистентов)
- **Память** — ассистент помнит факты о тебе между запусками. Скажи «запомни, что…» (или
  `/remember …`) — и в любом будущем разговоре он это учтёт. Хранится в
  `C:\Users\<ты>\.deepseek-assistant\memory.md`.
- **Сессии** — каждый разговор сохраняется автоматически (даже если закрыть окно). При запуске
  увидишь «Прошлых разговоров: N». `/sessions` — список, `/resume` — продолжить с того места,
  `/new` — начать заново.

## 4. Как понять, какая модель отвечает

Ассистент сам переключается между моделями, если у сильнейшей кончился лимит. Чтобы ты всегда
был в курсе:

- **В приглашении** слева видно активную модель: `[deepseek-v3-0324] ты ›`.
- **После каждого ответа** — строчка `— deepseek-v3-0324` (или `— gpt-4o`).
- **При переключении** — жёлтое предупреждение: `⚠ deepseek-v3-0324 занят (лимит) → перешёл на gpt-4o`.
- Команда **`/model`** покажет всю цепочку и текущую.

**Важно:** команды чата при любой модели **одинаковые** — переключение прозрачное, ничего
перенастраивать не надо.

Порядок моделей (от сильной к запасной):
1. `deepseek/deepseek-v3-0324` — основная, самая мощная;
2. `openai/gpt-4o` — запас №1, почти не уступает;
3. `openai/gpt-4o-mini` — запас №2, для простого, самый щедрый по лимитам.

## 5. PLAN MODE — как у «больших» ассистентов

Для рискованных задач (большие правки, удаление, незнакомый проект) сначала включи план:

```
/plan
перенеси всю логику работы с файлами в отдельный модуль
```
Ассистент изучит проект и выдаст **план по пунктам, ничего не трогая**. Если план нравится:
```
/run
ок, делай по плану
```
Теперь он выполнит правки, спрашивая подтверждение на каждое изменение (если не включён `/yolo`).

## 6. Как выжать максимум

- **Ставь конкретные задачи.** Не «улучши код», а «в `main.py` добавь обработку ошибки, если
  файл не найден, и выведи понятное сообщение».
- **Дай контекст.** Скажи, в каком файле/папке смотреть — будет быстрее и точнее.
- **Сложное — через `/plan`.** Сначала план, проверь, потом `/run`.
- **Доверяешь — включи `/yolo`,** чтобы не подтверждать каждый шаг (но следи за действиями).
- **`/reset` между разными задачами** — чтобы старый контекст не мешал.
- **Запускай из нужной папки** — ассистент работает с файлами и командами именно там.
- **Просто общайся.** Он не только про код: объяснит, посоветует, напишет текст, разберёт ошибку.

### Примеры задач
```
изучи этот проект и скажи, из чего он состоит
найди, где задаётся список моделей, и добавь туда ещё одну
почему падает скрипт? запусти его и разберись
напиши bash-скрипт, который архивирует папку logs с датой в имени
объясни простыми словами, что делает этот регэксп: ^\d+[.)]\s+
```

## 7. Честно про лимиты и качество

- **Лимиты (GitHub Models, бесплатно, примерно):** сильные модели (DeepSeek-V3, gpt-4o) —
  ~10 запросов/минуту и ~50/сутки; gpt-4o-mini — щедрее (~150/сутки). Один «ход» агента с
  правкой нескольких файлов тратит несколько запросов. Минутный лимит восстанавливается через
  минуту, суточный — раз в день. Встроенное авто-переключение сглаживает это.
- **Публичность репозитория на лимиты НЕ влияет** — они на твой аккаунт/токен.
- **Качество:** DeepSeek-V3 и gpt-4o — сильные, для большинства задач отличные. На самых
  сложных многошаговых задачах топовый платный Claude ещё немного впереди — но для создания и
  поддержки проектов этого ассистента более чем достаточно, и он бесплатный и без подписки.
